Bash can sort numbers pretty efficiently

I usually hang out in #bash on Freenode IRC. Bash is a really nice shell and the people in that channel are always friendly. Well, most of them are.

Playing around with arrays, I found this trick:

sortuniq () {
  for number
  do
    dumb[$number]=0
  done
  echo ${!dumb[@]}
}

It works like this:

$ sortuniq 1 3 1 6 2
1 2 3 6`

The output is the same as sort | uniq (or sort -u), hence the name.

This is a pseudo-sorting algorithm that runs in O(n) :)
It's basically a bucket sort, it works for integers from 0 to 2**63 -1.

It works like a charm for numbers. Then why not declaring dumb as an associative array and use it for strings too? Well, strings are a completely different matter. Associative arrays are hashed, the hashing function is like

i=0; for char in string; do ((i*=16777619, i^=ord(char) )); done

which happens to "work" when the string is only 1-char long.

That sorting function can be minified to this:

s()(for n;do d[$n]=;done;echo ${!d[@]})

Only 39 bytes. It's pretty cool, imho.

Apparently nobody knew about this, so... yeah...
